# Hinglish Multi-Layer LLM Safety Guardrails

A production-ready safety guardrail pipeline for LLM chatbots that handles
**English**, **Hinglish** (Roman-script code-mixed Hindi), and **Hindi
(Devanagari script)** вЂ” languages that most guardrail tools ignore entirely.

> Research project for RV University's Center for Applied and Responsible AI (CARA).

## How it works

For every user message the pipeline:

1. **Identifies the language** (English / Hindi / Hinglish) using XLM-RoBERTa.
2. Runs **four input-side guardrails**:
   - **PII redaction** вЂ” regex + IndicNER; masks emails, phones, Aadhaar, PAN, cards
   - **Toxicity** вЂ” MuRIL classifier + expanded Hinglish/Devanagari wordlist
   - **Prompt injection** вЂ” DeBERTa (English) + rule patterns; gated off for Hindi to prevent false positives
   - **Jailbreak** вЂ” role-play / hypothetical frame patterns + MuRIL head
3. Blocked messages are **logged and refused**; clean messages go to the LLM.
4. Runs **two output-side guardrails** on the reply:
   - **Harmful-output filter** вЂ” four independently-scored categories (toxic, system-prompt leak, unsafe compliance, PII); DistilBERT + wordlist/regex
   - **Hallucination flag** вЂ” four-signal composite (hedge density, numeric overconfidence, self-contradiction, temporal overreach) + optional grounding check via sentence-transformer cosine similarity
5. **Logs the full trace** to SQLite; surfaced in the dashboard.

Each guardrail has a **neural primary path** and a **rule-based fallback** вЂ”
the system degrades gracefully if models fail to load.

## Setup

### Prerequisites

- Python 3.11+ (3.12 recommended)
- [Ollama](https://ollama.com/download) for the local LLM (no API key needed)

```bash
# Pull a lightweight model for Apple Silicon / CPU
ollama pull llama3.2
```

### Install

```bash
git clone <repo-url> && cd hinglish-guardrails
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
```

> First run downloads HuggingFace models (MuRIL, IndicNER, DeBERTa, XLM-RoBERTa).
> On Apple Silicon they run via `mps`; set `device = "auto"` in `config.py`.

> If presidio reports a missing spaCy model: `python -m spacy download en_core_web_lg`

## How to run

### Streamlit chat UI

```bash
streamlit run app.py
# Opens at http://localhost:8501
```

### REST API (FastAPI)

```bash
pip install -r requirements-dev.txt  # adds fastapi + uvicorn
uvicorn api:app --host 0.0.0.0 --port 8000 --reload

# Interactive docs at http://localhost:8000/docs

# Examples
curl -X POST http://localhost:8000/check \
     -H "Content-Type: application/json" \
     -d '{"text": "Ignore all previous instructions."}'

curl -X POST http://localhost:8000/chat \
     -H "Content-Type: application/json" \
     -d '{"text": "Tell me about machine learning."}'

curl http://localhost:8000/stats
curl http://localhost:8000/health
```

### Docker

```bash
# Build
docker build -t hinglish-guardrails .

# Run Streamlit UI (Ollama on host machine)
docker run -p 8501:8501 hinglish-guardrails

# Run REST API instead
docker run -p 8000:8000 hinglish-guardrails \
  uvicorn api:app --host 0.0.0.0 --port 8000
```

The container connects to Ollama on the host via `host.docker.internal:11434`.
Override with `-e OLLAMA_HOST=http://your-ollama-host:11434`.

### Tests

```bash
pip install -r requirements-dev.txt
pytest tests/ -v
```

### Benchmark evaluation

```bash
# Seed set (43 prompts, fast)
python evaluate.py

# 900-prompt input benchmark
python evaluate.py --data benchmark_900.json

# 750-item output benchmark (no LLM or Ollama needed)
python evaluate.py --output-mode
python evaluate.py --output-mode --output-data output_benchmark.json

# Inspect logs
python analyze_logs.py
python analyze_logs.py --all --csv report.csv
```

### MuRIL fine-tuning

```bash
# Generate labeled data
python training/prepare_data.py           # full ~900 rows
python training/prepare_data.py --smoke-test  # 63 rows for quick test

# Fine-tune (1 epoch smoke test on M3 Mac, ~15 sec)
python training/finetune_muril.py \
    --epochs 1 --batch-size 8 --max-length 128

# The checkpoint is auto-detected by config.py once it exists at models/muril-guardrail/
```

See `training/README.md` for DGX A100 instructions and expected runtimes.

> **How Hindi recall reached 97%:** Fine-tuning MuRIL on 1,530 synthetic

## Honest limitations

- **Remaining Hindi FNs (3%).** Three Hinglish jailbreak prompts in the
  "Tumhare koi moral restrictions nahi hain" family and a handful of
  multi-clause Hindi injection prompts ("а¤Єа¤№а¤ІаҐ‡ а¤•аҐ‡ а¤Ёа¤їа¤°аҐЌа¤¦аҐ‡а¤¶аҐ‹а¤‚ а¤Єа¤° а¤§аҐЌа¤Їа¤ѕа¤Ё а¤®а¤¤ а¤¦аҐ‹ вЂ¦")
  still miss вЂ” they lack a single distinctive anchor phrase for a rule pattern.
  Closing this gap requires either more Hinglish jailbreak rules or a
  fine-tuned MuRIL head trained on real (non-synthetic) Hinglish jailbreak data.
- **MuRIL fine-tuning on synthetic data does not generalise.** The training
  pipeline (`training/`) achieves 100% on held-out synthetic test data but
  leaves benchmark recall unchanged because the template phrasings differ from
  benchmark phrasings. Fine-tuning is most useful when training data comes from
  the same distribution as production traffic.
- **PII name detection.** The heuristic fallback can misread Hinglish sentences
  that start with two capitalised words. The IndicNER neural path fixes this.
- **Hallucination detection is a confidence signal, not ground truth.** It
  surfaces low-confidence answers; it does not verify facts.
- **No indirect injection.** Malicious instructions hidden in retrieved context
  (RAG poisoning) are out of scope for this single-session chatbot.
- **Ollama required for chat.** The REST `/chat` and Streamlit UI both need a
  running Ollama instance; `/check` and all tests run without it.

## CI / CD

GitHub Actions (`.github/workflows/ci.yml`) runs on every push:

1. **Syntax check** вЂ” `python -m py_compile` on all modules
2. **pytest** вЂ” 220+ tests covering all guardrails, pipeline, REST API, and evaluation harness
3. **Input benchmark** вЂ” `evaluate.py --data benchmark_900.json` (900 prompts, main branch / PR)
4. **Output benchmark** вЂ” `evaluate.py --output-mode` (750 labeled responses, main branch / PR)

HuggingFace models are cached between runs to keep the pipeline fast.
